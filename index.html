The labs
How it works — add a lab in three steps
01

Run

run-lab.sh sends every task through every model via Claude Code (Max plan, no API key). Served model + tokens + cost come from billing telemetry — not self-reports.

02

Grade

grade.py turns raw runs into an auditable results.json — metrics only; the models' generated code is discarded.

03

Decide

Update routing.json and this hub picks it up. The recommendation is the point; the decks are the evidence.

Harness: /harness · everything here is reproducible.

Kriya Model Lab — an engineering artifact, not a research write-up. It exists to answer one operational question, repeatedly and with evidence: which model and effort tier should each Kriya agent role use?
